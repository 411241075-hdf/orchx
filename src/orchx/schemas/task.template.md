<result_file>
В самом конце работы запиши итог в **`{{result_path}}`** одним вызовом
встроенного `write` tool. Файл может ещё не существовать — `write` его создаст.

Формат (по схеме `orchx/schemas/result.schema.json`):

```json
{
  "task_id": "{{task_id}}",
  "status": "success",
  "artifacts": ["относительный/путь/к/изменённому/файлу"],
  "notes": "Что сделано, какие компромиссы. Если что-то не получилось — почему.",
  "metrics": {},
  "needs_followup": []
}
```

Допустимые `status`:

- `success` — все цели достигнуты, acceptance должны пройти.
- `partial` — сделана часть, остальное требует ручного решения (опиши в notes).
- `failed` — задача невыполнима как сформулирована (опиши блокер в notes).

**Запись в `{{result_path}}` обязательна** независимо от того, как пошла работа.
Без неё задача автоматически считается провалившейся.
</result_file>
